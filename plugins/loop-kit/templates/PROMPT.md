## 毎イテレーションの手順

1. **現状確認から始める**: git log と `<進捗ファイル: 例 .loop/<slug>/notes.md>` を読み、
   前のイテレーションで何が済んでいて何が失敗したかを把握する
2. **このイテレーションでやることを1つだけ選ぶ**(最重要ルール。
   残タスクのうち最も小さく検証可能なものを1つ。全部を一度にやろうとしない)
3. 実装する
4. **検証する**: `<検証コマンド: 例 npm test>` を実行し、結果を確認する
5. 検証がパスしたら `git add -A && git commit -m "loop: <やったこと>"` でコミットする
6. **学びを記録する**: 失敗・ハマり・次イテレーションへの申し送りを
   `<進捗ファイル>` に追記する(次のイテレーションの自分への手紙)

## 厳守事項

- 1イテレーション1タスク。欲張らない
- 検証がパスしないままコミットしない
- `<触ってはいけないファイル・ディレクトリがあれば列挙>`
- 同じ失敗を2回したら、3回目は別のアプローチを試し、その旨を進捗ファイルに書く

## 完了条件

以下がすべて真になったときだけ、`<promise>COMPLETE</promise>` を出力する:

- [ ] <条件1: 例 全 CRUD エンドポイントが動く>
- [ ] <条件2: 例 npm test が全件パス>
- [ ] <条件3: 例 README に使い方が書いてある>

**条件が偽のうちは絶対に完了宣言を出力しない。ループを抜けるために嘘をつかない。**

## 行き詰まったとき

<N: 例 15> イテレーション相当の試行をしても完了が見えない場合:
- 何がブロックしているかを進捗ファイルに文書化する
- 試したアプローチを列挙する
- 代替案を提案する
(その場合も完了宣言は出さない。上限到達で止まるのは正常な結果)
