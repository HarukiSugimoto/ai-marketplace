# pm-kit — 議事録・意思決定ログシステム

会議の雑メモを議事録に、重要な決定を ADR 風の記録に、未消化アイテムを次回アジェンダに
変換するプラグイン。PM 的に動くときの「言った言わない」「なぜああしたんだっけ」を潰す。

```
会議 ─→ /pm-minutes ─→ minutes/(決定・ToDo・持ち越し)
              │ 重要な決定は昇格
              ▼
        /pm-decision ─→ decisions/(ADR風・1件1ファイル)
              │
        /pm-agenda  ←─ 未消化ToDo・未決・proposed を機械的に拾う
              ▼
         次回会議 ─→ また /pm-minutes(ループ)
```

## スキル

| スキル | 役割 |
|---|---|
| `/pm-minutes` | 雑メモ・文字起こし → 議事録+アクションアイテム(担当・期限つき、捏造禁止) |
| `/pm-decision` | 決定を「背景・選択肢・決定・理由・再検討条件」で記録。supersede 対応 |
| `/pm-agenda` | 過去ログの未消化アイテムを走査して次回アジェンダを起案(決定議題にはゴール必須) |

## 案件ディレクトリの構成

中身(議事録・決定)は**案件ディレクトリ側**に貯まる。プラグインは型と文化だけ持つ。

```
<職場ディレクトリ>/
  .pm-kit/workspace.yaml     # company: <会社id> の紐付け
  <案件>/
    minutes/                 # <日付>_<会議名>.md
    decisions/               # <NNN>_<slug>.md(連番・消さない)
    agendas/                 # <日付>_<会議名>.md
```

## 設計方針

- **捏造禁止が最優先**: 素材に無い担当・期限・理由は `要確認` と書く。
  議事録の推定補完は事故のもと
- **アクションアイテムは機械可読**: `- [ ] 内容(担当: X, 期限: Y)` 形式に統一し、
  /pm-agenda がチェックボックスを走査して拾う
- **会社パック**(`companies/<id>/rules.yaml`)で文体・敬称・機密表示・共有文化を差し替え。
  work-slides と同じ転職対応の思想
- 議事録→スライド化が必要なときは work-slides の minutes テンプレに接続できる
