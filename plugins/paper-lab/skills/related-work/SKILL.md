# related-work — ノート群から関連研究章を蒸留する

/paper-read で貯めたノートを、**自分の研究の位置づけを主張する章**に組み上げる。
関連研究は先行研究の紹介文集ではない — 各段落が
「先行研究群の要約 → その限界 → 自分の研究がどう埋めるか」の型で claim に収束すること。

## Step 0: 素材の棚卸し

1. `.paper-lab/research.yaml` を解決し、claim を確認する
2. `<notes_dir>/` の全ノートの frontmatter(citekey / relation / tags)と
   「どんなもの?」「自分の研究との関係」節を読む
3. 棚卸し結果を見せる: relation 別の件数と、**穴の指摘**
   - `compete` が 0 件 → 「競合が無い」はほぼ調査不足。/paper-survey を提案
   - 有名どころの欠け・古い論文しか無い系統があれば正直に指摘する

## Step 1: 構成案(★関門1: グルーピングのレビュー)

ノートを 2〜4 の系統にグルーピングして構成案を見せる。軸の例:
課題設定別 / 手法系統別 / 時系列(発展史)。**自分の研究との対比が立つ軸を選ぶ。**

```markdown
## 2. 関連研究
### 2.1 <系統A>          ← vaswani2017attention, ... (n件)
    限界: ...  → 本研究との差分: ...
### 2.2 <系統B>          ← ... (n件)
    限界: ...  → 本研究との差分: ...
```

**ここで停止。** 系統の切り方・順序・どの論文をどこに置くかを直してから先へ。

## Step 2: ドラフト生成

- 出力先の形式を確認: 修論が LaTeX なら `.tex`(`\cite{citekey}`)、
  それ以外は `.md`(`[@citekey]`)。citekey は refs.bib のものだけを使い、
  **bib に無い文献を引用しない**(必要なら先に /paper-read を促す)
- 各段落の型を守る: 要約(2〜3文)→ 限界(1〜2文)→ 本研究との関係(1文)。
  relation が `compete` の論文は差分を具体に(データ・設定・主張のどこが違うか)
- ノートに書いていないことを本文で断定しない。ノートが薄くて書けない箇所は
  `% TODO: <ノート名> の検証節を読み直す` と残す
- 出力: `papers/drafts/related_work.tex`(or `.md`)

## Step 3: セルフチェック(★関門2)

生成後、自分で以下を検査して結果を報告する:

- [ ] すべての段落が claim への収束(本研究との関係)で終わっているか
- [ ] `compete` の全論文に差分の言及があるか
- [ ] 引用キーがすべて refs.bib に存在するか(機械的に照合する)
- [ ] 「〜が提案されている」の羅列になっていないか(3連続したら書き直し)

指摘があれば直し、承認を得て完了。ノートが増えたら再実行すれば章も育つ、と伝える。
